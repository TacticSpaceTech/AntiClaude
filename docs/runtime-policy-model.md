# Runtime Policy Model

AntiClaude Guard is a local-only alpha policy evaluator. It is not a hosted gateway and it does not claim production enforcement. The current contract is a small TypeScript SDK that returns one of three decisions for agent interactions:

- `allow`: continue the interaction.
- `block`: reject the interaction.
- `review`: stop automatic execution and route to human review.

## Surfaces

The evaluator supports three surfaces:

| Surface | Purpose |
| --- | --- |
| `prompt` | Incoming user input before forwarding to the target agent. |
| `tool-call` | Tool-call-like model output before execution. |
| `output` | Final model output before returning data to the caller. |

## Rule Shape

Policies are JSON or YAML files with a `rules` array. Each rule includes:

- `id`: stable rule id.
- `surface`: `prompt`, `tool-call`, or `output`.
- `action`: `allow`, `block`, `review`, or `deny` (`deny` maps to `block`).
- `severity`: `critical`, `high`, `medium`, or `low`.
- `reason`: why the rule exists.
- `matchedEvidence`: generated by the evaluator when a rule matches.
- `recommendedAction`: operational guidance.
- `match`: `contains`, `regex`, `toolNames`, and/or `arguments`.

Tool names support `*` wildcards. Argument constraints support `exists`, `equals`, `contains`, and `regex` over dotted paths.

See `docs/examples/policies/anticlaude.policy.yaml` for the committed example policy.

## Precedence

If several rules match, the evaluator chooses the strongest action:

1. `block`
2. `review`
3. `allow`

Within the same action, higher severity wins. The decision still includes all matched rules so a UI or audit replay can explain the lower-priority evidence too.

## SDK Entry Points

The engine exports:

- `loadGuardPolicy(filePath)`
- `validateGuardPolicy(policy)`
- `evaluateGuardPolicy(policy, input)`
- `DEFAULT_GUARD_POLICY`

Example:

```ts
import { DEFAULT_GUARD_POLICY, evaluateGuardPolicy } from '@anticlaude/engine'

const decision = evaluateGuardPolicy(DEFAULT_GUARD_POLICY, {
  surface: 'tool-call',
  toolCall: {
    name: 'refund_user',
    arguments: { userId: 'u_123', amount: 9999 },
  },
})
```
