# AntiClaude Report Schema v1

AntiClaude scan reports are local JSON artifacts. Version `1` is the first stable shape used by the CLI compare gates, Web report views, CI summaries, and replay tooling.

## Top-Level Contract

Required fields:

| Field | Type | Notes |
| --- | --- | --- |
| `reportVersion` | `1` | Compatibility gate. Unknown versions are rejected by `readScanReportJson`. |
| `id` | string | Stable report id for compare output. |
| `timestamp` | string | ISO timestamp generated when the scan completes. |
| `targetEndpoint` | string | Target URL used by the scan. |
| `target` | object | Adapter, endpoint, auth, timeout, payload, and model metadata. |
| `duration` | number | Total scan duration in milliseconds. |
| `results` | array | One result per payload variant attempt. |
| `score` | number | Overall score from `0` to `100`; higher is better. |
| `owaspCoverage` | array | Per-category pass/fail coverage. |
| `summary` | object | Count summary for payloads, attempts, breaches, blocked attempts, and errors. |
| `reproduction` | object | CLI command and config needed to reproduce the scan shape locally. |

Optional fields:

| Field | Type | Notes |
| --- | --- | --- |
| `suite` | object | Deterministic suite metadata: `name`, `description`, `seed`, `count`, `categories`, `severities`, `tags`, and `payloadIds`. |

## Target Metadata

`target.adapter` must be one of:

- `generic-json`
- `openai-chat`
- `anthropic-messages`
- `custom-json`

`target.hasAuthHeader` records whether an auth header was configured. The report must not store the secret value. Request evidence uses redacted headers generated by the engine.

## Result Contract

Each `results[]` item records both the security finding and enough local evidence to debug it:

| Field | Type | Notes |
| --- | --- | --- |
| `payloadId` / `payloadName` | string | Payload identity. |
| `category` / `owaspCategory` | enum | Agentic Security Initiative category id. |
| `severity` | enum | `critical`, `high`, `medium`, or `low`. |
| `prompt` | string | Attack prompt sent to the target. |
| `request` | object | POST request evidence: `method`, `url`, `adapter`, redacted `headers`, and `body`. |
| `response` / `fullResponse` | string | Response excerpt and full normalized response captured by the adapter. |
| `leaked` | boolean | Whether the attempt produced a breach. |
| `status` | enum | `breached`, `blocked`, or `error`. |
| `confidence` | number | Detector or judge confidence. |
| `confidenceSource` | enum | `detector`, `llm-judge`, or `none`. |
| `indicators` | string[] | Detector indicators that explain the confidence. |
| `strategy` | enum | Attack variant strategy such as `direct`, `encoding`, or `semantic`. |
| `generation` | number | Variant generation index. |
| `requestDuration` | number | Request duration in milliseconds. |

Optional result fields:

- `remediation`: human-readable fix guidance.
- `error`: request, adapter, or judge error text.
- `errorState`: stable error state such as `timeout` or `judge-unavailable`.
- `judgeVerdict`: `{ leaked, confidence, reasoning }` when an LLM judge made the final decision.

## Error States

Errors should be explicit but local-first:

- Target transport failures use `status: "error"`, `confidenceSource: "none"`, and an `errorState` such as `timeout`.
- LLM judge outages should not erase detector evidence. If the target response was still evaluated by deterministic detectors, keep the detector result and add `errorState: "judge-unavailable"`.
- Reports must not include raw API keys, bearer tokens, session cookies, or unrelated local environment values.

## Examples

Committed examples live in `docs/examples/reports/`:

- `baseline-safe.json`: blocked system-prompt extraction.
- `current-vulnerable.json`: breached system-prompt extraction.
- `error-report.json`: target timeout.
- `judge-unavailable.json`: detector-only result with judge unavailable evidence.

The engine validator is exported as `validateScanReport`. The compare loader uses `readScanReportJson`, which rejects unsupported `reportVersion` values and invalid v1 reports before calculating regressions.
